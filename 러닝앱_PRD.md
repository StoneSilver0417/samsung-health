# PRD: 러닝 기록 분석 & 동기부여 앱 (가칭: RunLog)

**버전**: v1.0
**작성일**: 2026-06-11
**플랫폼**: Android (Flutter) — 삼성헬스 데이터는 Health Connect 경유로만 접근 가능하므로 웹앱 불가
**확장 옵션**: Firebase 데이터 기반 웹 대시보드 (v3 이후)

---

## 1. 개요

### 1.1 배경
- Galaxy Watch 7 + 삼성헬스로 야간 러닝(21~22시, 강변 코스) 기록 중
- 삼성헬스의 기본 분석은 단편적이고, 장기 추세·성장 시각화·동기부여 장치가 부족함
- 현재 23일 러닝 프로그램 진행 중 (30분 연속 달리기 → 5km 목표)

### 1.2 목표
1. **체력 증진 가시화**: 같은 거리/코스에서 페이스·심박이 개선되는 추세를 한눈에 확인
2. **동기부여**: 나이키 런 클럽(NRC) 스타일의 업적/배지/스트릭 시스템
3. **멋진 데이터 시각화**: 보고 싶어지는 차트와 기록 카드

### 1.3 성공 지표 (개인용 기준)
- 주 3회 이상 러닝 유지 (스트릭 지표)
- 30분 연속 달리기 달성 → 5km 무정지 완주
- 동일 코스 평균 심박 5% 감소 또는 동일 심박 페이스 개선 (체력 증진의 정량 지표)

## 2. 사용자 & 시나리오

### 2.1 타겟 사용자
- 1차: 본인 (단일 사용자, 개인 프로젝트)
- 2차: 추후 가족/지인 공유 가능성 (멀티유저는 v2 이후)

### 2.2 핵심 시나리오
1. 러닝 후 귀가 → 앱 실행 → Health Connect에서 새 기록 자동 동기화
2. 오늘 기록 요약 카드 확인 (거리, 페이스, 심박존, 칼로리)
3. 새 업적 획득 시 축하 애니메이션 + 배지 부여
4. 주간/월간 추세 차트로 체력 향상 확인
5. 23일 프로그램 진행률 체크

## 3. 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 프레임워크 | Flutter (Dart) | 기존 Flutter 경험 활용 |
| 헬스 데이터 | `health` (pub.dev) + Android Health Connect | 삼성헬스 → Health Connect 동기화 필수 (삼성헬스 설정에서 활성화) |
| 백엔드 | Firebase (Firestore + Auth) | 무료 티어 내 운영 |
| 로컬 캐시 | Hive | 오프라인 조회, Firestore 읽기 절감 |
| 차트 | `fl_chart` | 커스터마이징 자유도 높음. 대안: `syncfusion_flutter_charts`(무료 커뮤니티 라이선스) |
| 상태관리 | Riverpod | |
| 애니메이션 | `lottie` (업적 획득 연출), `flutter_animate` | |

### 3.1 Health Connect에서 읽을 데이터 타입
- `WORKOUT` (운동 세션: 시작/종료, 타입=러닝)
- `DISTANCE_DELTA`, `STEPS`
- `HEART_RATE` (세션 내 시계열)
- `TOTAL_CALORIES_BURNED`
- `SPEED` (페이스 산출)

> 주의: Health Connect는 기본 30일 이전 데이터 조회에 제한이 있을 수 있음. 최초 동기화 시 권한 및 조회 범위 확인 필요. 동기화된 데이터는 Firestore에 영구 저장하여 자체 히스토리 구축.

## 4. 핵심 기능

### 4.1 데이터 동기화 (P0)
- 앱 실행 시 마지막 동기화 시점 이후의 러닝 세션 자동 가져오기
- 수동 새로고침 (pull-to-refresh)
- 세션별 상세: 거리, 시간, 평균/최고 심박, 페이스, 칼로리, 심박 시계열
- 중복 방지: 세션 UUID 기준 dedupe 후 Firestore 저장

### 4.2 기록 분석 (P0)
- **러닝 상세 화면**: km별 스플릿, 심박존 분포(Z1~Z5), 페이스 그래프
- **추세 분석**:
  - 주/월/연 단위 거리·횟수·시간 누적
  - 평균 페이스 추이 (이동평균선 포함)
  - 심박 효율 지표: 동일 페이스 대비 평균 심박 추이 (체력 증진의 핵심 지표)
- **개인 기록(PB)**: 최장 거리, 최장 시간, 최고 페이스(1km/5km), 최다 주간 거리

### 4.3 업적 시스템 (P0) — NRC 스타일
업적은 카테고리별 배지로 구성, 획득 시 전면 축하 화면 + Lottie 애니메이션.

| 카테고리 | 예시 업적 |
|---|---|
| 거리 마일스톤 | 첫 1km / 5km / 10km / 누적 50km / 100km / 500km |
| 연속성(스트릭) | 주 3회 달성, 2주 연속, 4주 연속, 월 10회 |
| 시간 | 첫 10분 무정지 / 20분 / 30분 무정지 (프로그램 목표와 직결) |
| 개인 기록 | PB 갱신 (거리/페이스별 별도 배지) |
| 특수 | 야간 러너 (21시 이후 10회), 비 오는 날 러닝, 새해 첫 러닝, 월간 개근 |
| 프로그램 | 23일 프로그램 단계별 완료 배지 |

- 배지 보관함: 획득/미획득(실루엣 처리) 그리드, 진행률 표시 (예: 누적 100km까지 73km)
- 획득 조건은 로컬에서 매 동기화 시 평가 (서버 로직 불필요 → Firebase 무료 티어 유지)

### 4.4 데이터 시각화 (P0)
- **홈 대시보드**:
  - 이번 주 요약 링 (목표 대비 거리/횟수, 애플 액티비티 링 스타일)
  - 최근 러닝 카드 (지도 미니멀 표시는 v2, 좌표 데이터 가용성 확인 필요)
- **캘린더 히트맵**: GitHub 잔디 스타일, 일별 러닝 강도 색상화
- **추세 차트**: 라인(페이스/심박), 바(주간 거리), 누적 영역(연간 거리)
- **공유 카드**: 러닝 1회를 인스타 스토리 비율 이미지로 렌더링 (거리/페이스/배지 포함, v1)
- 디자인 톤: 다크 모드 기본 (야간 러너), 네온 포인트 컬러 1~2색, 숫자는 큰 타이포

### 4.5 23일 프로그램 트래커 (P1)
- 기존 프로그램(걷기/달리기 인터벌 → 30분 연속)을 앱 내 체크리스트로 내장
- 당일 미션 표시 + 완료 시 자동 체크 (해당일 러닝 세션 존재 여부로 판단)
- 프로그램 진행률 바, 완료 시 전용 배지

### 4.6 목표 설정 (P1)
- 주간 목표 (거리 또는 횟수)
- 월간 챌린지 (예: 6월 40km)
- 목표 대비 진행률 위젯

## 5. 데이터 모델 (Firestore)

```
users/{uid}
  └ profile: { nickname, createdAt, weeklyGoalKm, weeklyGoalCount }

users/{uid}/runs/{runId}            // runId = HealthConnect 세션 UUID
  { startTime, endTime, distanceM, durationSec,
    avgHr, maxHr, avgPaceSecPerKm, calories,
    splits: [{km, paceSec, avgHr}],
    hrSeries: [...],                // 용량 크면 압축 또는 다운샘플링
    syncedAt }

users/{uid}/achievements/{badgeId}
  { earnedAt, runId }

users/{uid}/stats/summary           // 집계 캐시 (읽기 절감)
  { totalKm, totalRuns, totalSec, currentStreak, longestStreak,
    pb: { longestKm, best1kPace, best5kPace }, updatedAt }
```

- 쓰기 시 `stats/summary`를 클라이언트에서 함께 갱신 (Cloud Functions 없이 무료 티어 유지)
- `hrSeries`는 1분 단위 다운샘플링하여 문서 1MB 제한 회피

## 8. 비기능 요구사항

- **비용**: Firebase 무료 티어 내 운영 (Functions 미사용, 집계는 클라이언트, Hive 캐시로 읽기 최소화)
- **프라이버시**: 건강 데이터는 본인 계정 Firestore에만 저장, 보안 규칙으로 uid 일치 강제
- **오프라인**: Hive 캐시로 네트워크 없이 조회 가능
- **성능**: 차트 렌더링 시 데이터 다운샘플링 (포인트 200개 이하)
